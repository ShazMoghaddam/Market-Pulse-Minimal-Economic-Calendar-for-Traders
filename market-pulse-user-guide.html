01  /  Quick Start

Up and running in under a minute

Market Pulse is a single HTML file. There is no installation, no terminal, no npm install. If you can open a file in a browser, you're ready to go.

Open Locally

  1. Download market-pulse.html from your Gumroad purchase.
  2. Double-click the file — it opens in your default browser.
  3. That's it. The calendar loads with built-in sample data immediately.
ℹ No internet needed to open it. Market Pulse includes fallback event data so it always shows something useful, even without a live API connection.

02  /  Live Data

Connecting to a live data feed

For real-time economic events, Market Pulse connects to the TradingEconomics API. A free-tier key is sufficient for personal use.

Step 1 — Get a free API key

  1. Go to tradingeconomics.com/analytics/api.aspx and create a free account.
  2. After signing in, navigate to your dashboard and copy your API key.

Step 2 — Add your key to Market Pulse

  1. Right-click market-pulse.html and choose Open With → Text Editor (Notepad on Windows, TextEdit on Mac, or any code editor).
  2. Press Ctrl+F (or ⌘+F on Mac) and search for: YOUR_API_KEY_HERE
  3. Replace that placeholder with your actual key:
// Before:
const API_KEY = 'YOUR_API_KEY_HERE';

// After (example):
const API_KEY = 'guest:guest';  // ← your real key goes here
  1. Save the file (Ctrl+S / ⌘+S) and refresh your browser.
✓ You'll know it's working when the top-right corner shows a green LIVE badge instead of OFFLINE.
⚠ Keep your API key private. Don't share your edited HTML file publicly (e.g., posting it on a forum). Anyone who sees the file can read your key. If you self-host, use environment variables or a server-side proxy instead.

Understanding event status

Each event card displays one of three statuses:

StatusWhat it means
LIVE The event is happening right now (within the release window).
UPCOMING Scheduled for later today or in coming days. A countdown timer is shown.
RELEASED Data has been published. Actual vs. forecast figures are displayed if available.

Reading an event card

Each card shows the event name, the country/region flag, the scheduled time in your local timezone, and the impact level:

HIGH   Fed decisions, NFP, CPI — major market movers.
MED   Important but secondary indicators.
LOW   Minor data releases, regional figures.

Once released, the card updates to show the actual result alongside the prior reading and market forecast where available.

04  /  Filtering

Cutting through the noise

The filter bar at the top of the calendar gives you full control over what you see. Filters can be combined freely.

FilterOptionsTip
Impact All / High / Medium / Low Set to High on busy days to stay focused.
Region All, US, EU, UK, JP, CA, AU, CN, NZ, CH Pair with your trading session for a clean view.
Status All / Upcoming / Released Use Upcoming to only see what's ahead of you.
💡 Trader workflow tip: At the start of your session, set Impact → High and Status → Upcoming. You'll see only the events that could move your positions before the end of the day.

06  /  Hosting

Accessing it anywhere

Market Pulse works in three different ways depending on how technical you are.

Option A — Local file (recommended for most)

Keep the file on your computer and open it in Chrome, Firefox, or Edge. Works immediately, no internet needed for the app itself (only for live data).

Option B — Netlify Drop (free, 2 minutes)

  1. Go to app.netlify.com/drop in your browser.
  2. Drag and drop your market-pulse.html file onto the page.
  3. Netlify gives you a public URL instantly. Bookmark it or share it.

Option C — GitHub Pages (free, slightly more setup)

  1. Create a free GitHub account at github.com.
  2. Create a new public repository (e.g., my-market-pulse).
  3. Upload market-pulse.html and rename it to index.html.
  4. Go to Settings → Pages → Source: main branch → Save.
  5. Your calendar is live at yourusername.github.io/my-market-pulse.
⚠ If hosting publicly: Remove your API key from the file before uploading it to any public repository or sharing the URL. Anyone who can view the page source can see the key. Use a server-side proxy if you want both security and live data on a public host.

07  /  FAQ

Troubleshooting & common questions

The status badge says OFFLINE. What do I do?

This means the app couldn't reach the TradingEconomics API. Check that (1) your API key is correctly pasted with no extra spaces, (2) you have an internet connection, and (3) your TradingEconomics account is active. The calendar will still show fallback data while offline.

The times look wrong — they don't match my timezone.

Market Pulse reads your timezone from your browser automatically. If times look off, check your operating system's clock and timezone setting. On Windows: Settings → Time & Language → Date & Time. On Mac: System Settings → General → Date & Time.

Can I use this on my phone or tablet?

Yes — the layout is fully responsive. On mobile, upload the file to a free host (like Netlify Drop) and visit the URL in your mobile browser, or use a file browser app that supports opening HTML files locally.

Can I share this file with a friend or colleague?

Yes, but first remove your API key from the file. Each person using live data should obtain their own free key from TradingEconomics. The MIT License allows free redistribution of the app itself.

Events from yesterday are still showing. How do I refresh?

Press Ctrl+Shift+R (Windows/Linux) or ⌘+Shift+R (Mac) for a hard refresh. This clears cached data and forces a fresh API request.

I accidentally broke the file while editing. What now?

Re-download the original market-pulse.html from your Gumroad purchase page — your order is permanent and you can download it as many times as you need. Then add your API key again from scratch.

Does this work in Safari?

Yes, Safari 14 and later is fully supported. If opening a local file, Safari may show a security warning — click "Allow" or consider using Chrome/Firefox for local files.
